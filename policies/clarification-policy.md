# Clarification Policy

The plugin should feel automatic, but not reckless.

## Ask Immediately

Ask before continuing when:

- Company name is missing.
- User asks for email/proposal but the offer is unknown.
- User asks for final proposal but pricing or budget range is unknown.
- User asks to send/write/update but has not approved the exact action.

## Ask Later

Continue first, then ask at the end when:

- Contact name is missing but persona is clear.
- Deal stage is unclear but likely from context.
- Public data is thin.
- Case study is missing.
- Preferred tone is unspecified.

## Question Style

- Ask 1-3 questions maximum.
- Make questions concrete.
- Explain what the answer unlocks.

Example:

```markdown
Minh co the tao brief ngay, nhung de viet email/proposal dung duoc thi can them:
1. Minh dang ban san pham/dich vu nao cho ho?
2. Co pricing hoac budget range chua?
```

## CTA Default

Do not ask for CTA in the default clarification flow unless the user clearly wants a different
action. If CTA is missing, default to a low-friction discovery or review call.

## Assumptions

When proceeding with assumptions:

- Put them in `Assumptions`.
- Do not hide them inside confident language.
- Lower confidence if assumptions affect sales strategy.
