# Proposal Section Templates

Use these sections to create proposals that feel specific instead of generic.

Use `[[EDIT: ...]]` for any placeholder the user should review.

## Cover

```markdown
# [[EDIT: outcome]] Proposal for [[EDIT: company]]
Prepared for: [[EDIT: contact]], [[EDIT: title]]
Prepared by: [[EDIT: rep / company]]
Date: [[EDIT: date]]
```

## Executive Summary

```markdown
[[EDIT: company]] is currently [[EDIT: specific sourced situation: scaling sales team, entering new market, improving
conversion, reducing manual work]].

The main opportunity is to [[EDIT: business outcome]] while reducing [[EDIT: pain / inefficiency]].

We recommend [[EDIT: solution/package]] because it directly addresses [[EDIT: pain point 1]] and [[EDIT: pain point 2]].
```

## Understanding Your Situation

Use 3-5 bullets only.

```markdown
- Your team is likely dealing with [[EDIT: pain]], based on [[EDIT: evidence/source]].
- [[EDIT: stakeholder/persona]] appears to care most about [[EDIT: priority]].
- The current process may be slowed by [[EDIT: manual step/tool gap]].
```

## Proposed Solution

```markdown
We propose [[EDIT: package/product/service]].

This includes:
- [[EDIT: deliverable 1]]
- [[EDIT: deliverable 2]]
- [[EDIT: support/onboarding]]

This does not include:
- [[EDIT: exclusion 1]]
- [[EDIT: exclusion 2]]
```

## Proof

Use real proof when available:

- Similar customer case study.
- Measured before/after result.
- Testimonial.
- Internal benchmark.

If no proof exists, use methodology proof:

```markdown
Our approach works because it standardizes the repeatable parts of sales preparation: account
research, message framing, proposal structure, and follow-up timing. The sales rep keeps control
of judgment and relationship management.
```

## Investment

Do not invent pricing. Use one of these formats:

```markdown
Recommended package: [[EDIT: package]]
Investment: [[EDIT: price or range]]
Billing: [[EDIT: monthly/annual/project]]
Assumptions: [[EDIT: seats, usage, scope]]
```

If pricing is missing:

```markdown
Pricing placeholder: [[EDIT: needs confirmation]]
To finalize this section, confirm seat count, package, and contract term.
```

## ROI

Use transparent assumptions:

```markdown
Assumption:
- [[EDIT: N]] sales reps
- [[EDIT: X]] hours saved per rep per week
- [[EDIT: $Y]] loaded hourly cost

Annual value = [[EDIT: N]] x [[EDIT: X]] x [[EDIT: $Y]] x 52 = [[EDIT: $Z]]
```

