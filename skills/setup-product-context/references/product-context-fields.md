# Product Context Fields

Use this structure when preparing persistent product context.

## Required Minimum

- Company name.
- Website or `Not set yet`.
- Product/service name.
- One-sentence description.
- Ideal customer profile.
- Starting price or package range.
- Default CTA.

## Strongly Recommended

- Poor-fit customer profile.
- Core use cases.
- Differentiators.
- Packages table.
- Verified proof points.
- Assumption-only proof points.
- Common objections and responses.
- Proposal defaults.
- Implementation timeline.
- Success metrics.

## Proof Point Rules

Classify each proof point as one of:

- `Verified`: backed by a customer, CRM record, case study, analytics, or source.
- `Internal capability`: true because the product can demonstrably do it.
- `Assumption`: plausible but not verified yet.
- `Target outcome`: a goal, not a claim.

Only `Verified` and `Internal capability` proof may appear in buyer-facing copy without a caution
label. `Assumption` and `Target outcome` can guide internal thinking, but must not be phrased as a
guaranteed result.
