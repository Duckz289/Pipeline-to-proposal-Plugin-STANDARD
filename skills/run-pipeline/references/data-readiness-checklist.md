# Data Readiness Checklist

Use this checklist before each artifact. It prevents the pipeline from failing when different
companies have different data quality.

## Prospect Brief

Required:

- Company name.

Helpful:

- Website/domain.
- Industry.
- Company size.
- CRM status.
- Recent trigger.
- Decision maker or target persona.
- ICP fit signal.
- Connector/source health.

If only company name exists, create a draft brief and mark missing fields.

## Qualification

Required:

- Company name.
- User's product/service or ICP context.

Helpful:

- Industry.
- Company size.
- Geography.
- Current CRM stage.
- Known pain or trigger.
- Budget or buying motion.

If ICP context is missing, ask the smart-intake questions before scoring. If company data is thin,
mark fit as `unknown` rather than guessing.

## Outreach Email

Required:

- Company name.
- Product/service or offer.
- Sales stage: cold, follow-up, post-call, proposal follow-up, re-engagement.
- Recipient or target persona.

Helpful:

- Prospect brief.
- Specific pain signal.
- Desired CTA.
- Similar customer proof.

If recipient is unknown, draft for a persona and label it "persona-based draft".

## Proposal

Required for final proposal:

- Company name.
- Product/service.
- Pain points.
- Scope or package.
- Pricing or budget range.
- Next step.

If pricing or scope is missing, produce a proposal skeleton and ask for the missing fields.

## Next Best Action

Required:

- Company/contact.
- Last known interaction or explicit statement that no history is available.

Helpful:

- CRM timeline.
- Deal stage.
- Last email date.
- Proposal sent date.
- Stakeholder map.
- Email opens, link clicks, proposal views, or other tracking data if the user provides it.

If no timeline exists, produce a default cold outreach or re-engagement path.

## CRM Note Draft

Required:

- Summary of what was learned or drafted.
- Human approval before writing.

Helpful:

- Deal ID/contact ID.
- Next task date.
- Owner.

If IDs are unavailable, produce note text only.
